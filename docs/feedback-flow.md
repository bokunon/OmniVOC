# OmniVOC - フィードバック処理フロー

## 全体の流れ

```
[各プロダクト]                              [OmniVOC]

Web: <script> ウィジェット ─────┐
Slack Bot (既存): API 転送 ─────┤
LINE Bot (既存): API 転送 ──────┤──→ POST /api/feedback ──→ DB に蓄積
その他: API 転送 ───────────────┘              │
                                              ▼
                                      管理ダッシュボード
                                              │
                                    ┌─────────┼─────────┐
                                    ▼         ▼         ▼
                               チケット化   紐付け    スルー
                                    │         │         │
                                    ▼         ▼         ▼
                               ticketed   ticketed   resolved
```

## フィードバックのライフサイクル

```
                    ┌─────────────────────────────┐
                    │     フィードバック受信        │
                    │  POST /api/feedback          │
                    └──────────────┬──────────────┘
                                   │
                                   ▼
                          ┌────────────────┐
                          │  ステータス:    │
                          │    new          │
                          └───────┬────────┘
                                  │
                    ┌─────────────┼─────────────┐
                    │             │             │
                    ▼             ▼             ▼
             ┌──────────┐ ┌──────────┐ ┌──────────────┐
             │ チケット化 │ │ 既存に   │ │ 解決済みに   │
             │ (新規     │ │ 紐付け   │ │ マーク       │
             │  Issue)   │ │          │ │              │
             └─────┬────┘ └─────┬────┘ └──────┬───────┘
                   │            │              │
                   ▼            ▼              ▼
            ┌───────────┐ ┌───────────┐ ┌───────────┐
            │ ticketed   │ │ ticketed   │ │ resolved   │
            │ + issue_url│ │ + issue_url│ │ (+ issue   │
            │            │ │            │ │   url 任意)│
            └───────────┘ └───────────┘ └───────────┘
```

## ステータス定義

| ステータス | 意味 | 遷移条件 |
|-----------|------|---------|
| `new` | 受信済み・未処理 | フィードバック受信時に自動設定 |
| `ticketed` | チケット化済み | 新規 Issue 作成 or 既存 Issue に紐付け |
| `resolved` | 解決済み | 対応済みの同種フィードバックとしてスルー |

## 後続フィードバックの判断フロー（管理者の手動判断）

```
新しいフィードバックが来た
        │
        ▼
既存チケットと同じ内容か？ ──── NO ───→ そのまま new で待機
        │                                 → いずれチケット化
       YES
        │
        ├── チケットが open ──→ 「既存チケットに紐付け」
        │                       (要望の +1 として記録)
        │
        └── チケットが closed ──→ 「解決済み」にマーク
                                   (対応済みなのでスルー)
```

## V2（将来）: AI 補助の導入ポイント

| 導入箇所 | 内容 |
|---------|------|
| 類似判定 | 新規フィードバック受信時に、既存フィードバック / Issue との類似度を計算 |
| グルーピング提案 | 未処理フィードバックの中から似たものをクラスタリング |
| 自動ステータス提案 | 「この Issue (#42) と同じ内容では？」とサジェスト |

V1 では全て手動判断。量が増えてきたタイミングで AI 補助を段階的に導入する。
